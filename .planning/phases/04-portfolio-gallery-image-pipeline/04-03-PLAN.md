Astro getImage() API (verified at https://docs.astro.build/en/guides/images/#generating-images-with-getimage):

```typescript
import { getImage } from 'astro:assets';
import hero from '@/assets/placeholder/hero.jpg';

const heroAvif = await getImage({
  src: hero,
  widths: [640, 1024, 1536, 1920, 2400],
  formats: ['avif'],
  format: 'avif',  // primary output
});
// heroAvif.srcSet.attribute -> "/_astro/hero.HASH1.avif 640w, /_astro/hero.HASH2.avif 1024w, ..."
// heroAvif.attributes.srcset -> same (depending on Astro version)
// heroAvif.src -> default fallback URL
```

The URLs include the same content hashes Astro emits when `<Picture>` runs — they MUST match for the browser to deduplicate the fetch.
