    ## Step 3: SPF MERGE CHECK (critical — read before adding TXT)

    Before adding the Resend SPF TXT record, check Phase 1 DNS snapshot for an EXISTING SPF record on `klphotography.ie`.

    Run:
    ```
    grep -i "spf1\|TXT" .planning/phases/01-foundation-dns-preflight/<actual-snapshot-filename>
    ```

    **Case A: No existing SPF record found in snapshot.**
    -> Add the Resend SPF TXT record exactly as shown by Resend's wizard. Continue to Step 4.

    **Case B: Existing SPF record found** (likely format `v=spf1 include:_spf.google.com ~all` or similar from Wix/Google Workspace).
    -> DO NOT add a second TXT SPF record. Multiple SPF records on the same host cause a PermError and ALL SPF records get ignored (Pitfall 5 from Phase 5 RESEARCH).
    -> Instead, MERGE the includes into one record. Example:
       - Existing: `v=spf1 include:_spf.google.com ~all`
       - Resend wants: `v=spf1 include:_spf.resend.com ~all`
       - Merged (add to CF DNS as the single SPF record): `v=spf1 include:_spf.google.com include:_spf.resend.com ~all`
    -> Use the merged record. Save in CF DNS dashboard as a TXT record on the root.

    Sanity check the merged record:
    ```
    dig +short klphotography.ie TXT
    ```
    -> Expect exactly ONE line starting `v=spf1` and ending `~all`. If you see two `v=spf1` lines, delete one in CF DNS and try again.
