<objective>
Establish the design token layer (colors, typography, spacing, breakpoints) inside `src/styles/global.css` via Tailwind v4's `@theme` directive, wire Astro 6's built-in Fonts API to self-host EB Garamond + Inter via Fontsource (GDPR-safe, no Google CDN), and apply the editorial type scale (clamp-based headings, mobile→`sm:` body step, reduced-motion blanket rule) in `@layer base`.

Purpose: Phase 1 left `global.css` as a single `@import "tailwindcss";` line and made the explicit decision (per `01-01-SUMMARY.md`) that Tailwind v4 tokens live in CSS. This plan fills that hole so Plan 02-02 (primitives + styleguide) can consume `bg-cream`, `text-ink`, `font-serif`, and clamp-scaled headings without any further token work. After this plan, every later phase (03 content, 04 gallery, 05 form) inherits the editorial visual language automatically.

Output: Extended `src/styles/global.css` with `@theme` + `@layer base`, extended `astro.config.mjs` with `fonts[]` array, extended `src/layouts/BaseLayout.astro` with `<Font />` head injections.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| build-time → CDN | Astro Fontsource provider fetches `.woff2` files from `cdn.jsdelivr.net/npm/@fontsource/*` during `npm run build`; outputs hashed assets into `dist/_astro/fonts/`. After build, no runtime third-party requests. |
| visitor → CF Pages | Compiled CSS + woff2 served from Cloudflare Pages static origin. No GDPR-relevant data exchange (no Google Fonts CDN, no cookies). |
| user CSS preferences → page | `prefers-reduced-motion` MediaQuery flows from browser to page; handled purely in CSS, no JS. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Information disclosure | Font loading | mitigate | Self-host via Astro Fonts API; no `<link>` to fonts.googleapis.com or fonts.gstatic.com at runtime (verified via `grep -r 'googleapis\|gstatic' dist/` returning empty). Visitor IP never reaches Google → GDPR-safe (per CLAUDE.md EU/IE constraint cited in RESEARCH.md). |
| T-02-02 | Tampering | Font CDN at build time | accept | Fontsource is sourced from `cdn.jsdelivr.net/npm/@fontsource/*`, which mirrors the public npm registry. Compromise vector is the same as any npm dep; risk is low for two extremely popular open-source font packages. Mitigation deferred to standard supply-chain hygiene (lockfile + dep updates). |
| T-02-03 | Denial of service | Build-time font download | accept | First build is ~5–10s slower as Astro fetches woff2 from jsdelivr; subsequent builds use the on-disk cache. If jsdelivr is unreachable at build time, build fails fast — surfaces clearly, no silent regression. Fallback path documented in RESEARCH.md Environment Availability (hand-rolled `@font-face` from `public/fonts/`). |
| T-02-04 | Denial of service | Reduced-motion abuse via CSS | mitigate | Blanket `@media (prefers-reduced-motion: reduce) { *, ::before, ::after { animation-duration: 0.01ms !important; ... } }` rule neutralizes any animation that ships without explicit `motion-reduce:` modifier. Prevents the "missed-modifier → vestibular harm" failure mode (RESEARCH.md §7 + Pitfall 5). |
| T-02-SC | Tampering | npm installs | accept | This plan introduces NO new npm packages (RESEARCH.md Standard Stack: "Net-new in this phase: None"). All deps already approved in Phase 1. Package Legitimacy Gate not triggered. |
</threat_model>
