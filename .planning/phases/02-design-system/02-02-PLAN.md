**B. Create `src/components/ui/Button.astro`** per RESEARCH.md §5 worked example (per D-07: plain `.astro` files with `interface Props extends HTMLAttributes<...>` + `<slot />` + `class:list`; NO `tailwind-merge`, NO CVA).

Frontmatter:
- Import: `import type { HTMLAttributes } from 'astro/types';`
- Types: `type Variant = 'primary' | 'secondary' | 'ghost';` and `type Size = 'sm' | 'md' | 'lg';`
- Props: `interface Props extends HTMLAttributes<'button'> { variant?: Variant; size?: Size; as?: 'button' | 'a'; href?: string; loading?: boolean; }`
- Destructure: `const { variant = 'primary', size = 'md', as = 'button', href, loading = false, disabled, class: className, ...rest } = Astro.props;`
- `const Tag = as;`
- `const isInert = loading || disabled;` (loading must imply disabled per Phase 5 contact-form needs)
- Base array (with motion-safe transition per D-04 — this is the styleguide's motion demo; see Task 2 for how it's surfaced):
  - `'inline-flex items-center justify-center gap-2'` (gap so spinner sits next to label)
  - `'font-sans font-medium tracking-wide'`
  - `'rounded-none border'` (editorial = sharp corners)
  - `'motion-safe:transition-colors motion-safe:duration-150'` (the ONE motion example per D-04)
  - `'focus-visible:outline-2 focus-visible:outline-offset-4 focus-visible:outline-bronze'`
  - `'disabled:opacity-50 disabled:pointer-events-none'`
  - `'aria-busy:cursor-progress'` (visual cue while loading)
- `variants` record: primary = `'bg-ink text-cream border-ink hover:bg-bronze-hover hover:border-bronze-hover'`; secondary = `'bg-transparent text-ink border-ink hover:bg-ink hover:text-cream'`; ghost = `'bg-transparent text-ink border-transparent hover:text-bronze'`.
- `sizes` record: sm = `'px-4 py-2 text-sm'`; md = `'px-6 py-3 text-base'`; lg = `'px-8 py-4 text-base sm:text-lg'`.

Template:
```astro
<Tag
  href={Tag === 'a' ? href : undefined}
  disabled={isInert || undefined}
  aria-busy={loading ? 'true' : undefined}
  class:list={[...base, variants[variant], sizes[size], className]}
  {...rest}
>
  {loading && (
    <span
      class="inline-block h-3 w-3 rounded-full border-2 border-current border-r-transparent motion-safe:animate-spin"
      aria-hidden="true"
    ></span>
  )}
  <slot />
</Tag>
```

The spinner is a CSS-only span (border trick, `border-r-transparent` for the arc, `motion-safe:animate-spin` so reduced-motion users see a static ring instead of rotation). No SVG, no client JS, no extra dependencies. Phase 5 contact form will pass `loading` while the form submission is in flight; current styleguide will demo it (Task 2 #buttons section).
