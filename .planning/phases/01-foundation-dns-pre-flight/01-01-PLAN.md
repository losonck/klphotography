  <files>package.json, astro.config.mjs, src/styles/global.css, src/layouts/BaseLayout.astro, src/pages/index.astro</files>
  <read_first>package.json, astro.config.mjs, src/layouts/BaseLayout.astro, src/pages/index.astro</read_first>
  <action>
Install Tailwind v4 the modern way — through the Vite plugin, not the legacy `@astrojs/tailwind` integration. Tailwind v4 is configured in CSS, not in `tailwind.config.js`.

- `npm install -D tailwindcss@^4 @tailwindcss/vite@^4`

- Update `astro.config.mjs` to register the Vite plugin:
  ```js
  import { defineConfig } from 'astro/config';
  import tailwindcss from '@tailwindcss/vite';

  export default defineConfig({
    site: 'https://klphotography.ie',
    output: 'static',
    image: { service: { entrypoint: 'astro/assets/services/sharp' } },
    vite: { plugins: [tailwindcss()] },
  });
  ```

- Create `src/styles/global.css`:
  ```css
  @import "tailwindcss";
  ```
  Leave room — Phase 2 design system extends this file with `@theme` tokens. Do NOT define palette/typography tokens here yet.

- Import the stylesheet from `src/layouts/BaseLayout.astro`:
  ```astro
  ---
  import '@/styles/global.css';
  // existing frontmatter
  ---
  ```

- Update `src/pages/index.astro` to apply two sample Tailwind utility classes so we can sanity-check that Tailwind is wired:
  - `<h1 class="text-4xl font-bold tracking-tight">KL Photography</h1>`
  - Wrap content in `<main class="min-h-screen bg-stone-50 text-stone-900 p-8">…</main>`

Do NOT create a `tailwind.config.js` — Tailwind v4 reads `@theme` from CSS. Confirm by running `npm run build` and inspecting `dist/_astro/*.css` for the utility classes used above.
  </action>
